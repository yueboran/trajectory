# 璀璨星图 (迹向) - 生产环境部署指南

本文档提供了将本项目（React/Vite 前端 + FastAPI 后端）正式推向生产环境服务器的常规部署流程。推荐使用 Nginx 作为反向代理服务器和前端静态文件服务器。

## 🌟 核心部署架构图

在服务器上，流量的流向通常是这样的：

```text
用户浏览器 ──> [域名:80/443] ──> Nginx 服务器 (负责分发请求)
                                 ├──> [路径: /] 返回 Vite 打包好的前端静态 HTML/JS/CSS 文件
                                 └──> [路径: /api] 代理转发到本机的 Python FastAPI (如: localhost:8000)
```

---

## 第一步：准备服务器与环境

您需要一台 Linux 服务器（推荐 Ubuntu 20.04/22.04 或 CentOS）并安装基础环境：

1. **安装 Nginx**：用于提供 Web 服务和反向代理。
2. **安装 Node.js**（可选）：如果你在本地打包前端，服务器不需要 Node；如果要在服务器上直接打包代码，则需要安装。
3. **安装 Python 3.10+ 和 uv**：用于运行后端的 FastAPI 项目。

---

## 第二步：前端打包与部署 (Vite + React)

前端在生产环境中**不需要运行 `npm run dev`**，而是要编译成纯静态文件。

1. **配置生产 API 地址**：确保前端代码中请求后端的 URL 是相对路径（如 `/api/projects`），或者在环境配置文件（如 `.env.production`）中正确配置了生产环境的 API 域名。
2. **打包前端项目**：在本地的 `frontend` 目录下执行：
   ```bash
   npm install
   npm run build
   ```
3. **上传到服务器**：执行完毕后，会生成一个 `dist` 文件夹。将这个 `dist` 文件夹上传到服务器的一个目录下，例如 `/var/www/starmap/frontend/dist`。

---

## 第三步：后端环境配置与运行 (FastAPI)

后端不能直接用简单的 `python run.py`，在生产环境中为了处理高并发和保证进程稳定，通常需要用到 **Uvicorn + Gunicorn** 组合，或使用后台守护进程（如 Systemd）。

1. **上传后端代码**：将 `backend` 目录上传到服务器（如 `/var/www/starmap/backend`）。
2. **安装依赖**：
   ```bash
   cd /var/www/starmap/backend
   uv venv
   source .venv/bin/activate
   uv pip install -r requirements.txt
   # 或者直接安装必备依赖:
   uv pip install fastapi uvicorn
   ```
3. **后台持久化运行 (使用 Systemd)**：
   在服务器上，推荐使用 `Systemd` 来将后端注册为一个系统服务，这样即使服务器重启它也会自动启动。
   创建服务文件：`sudo nano /etc/systemd/system/starmap-backend.service`

   ```ini
   [Unit]
   Description=StarMap FastAPI Backend
   After=network.target

   [Service]
   User=root
   WorkingDirectory=/var/www/starmap/backend
   Environment="PATH=/var/www/starmap/backend/.venv/bin"
   # 生产环境启动命令
   ExecStart=/var/www/starmap/backend/.venv/bin/uvicorn run:app --host 127.0.0.1 --port 8000
   Restart=always

   [Install]
   WantedBy=multi-user.target
   ```
   启动后端服务：
   ```bash
   sudo systemctl start starmap-backend
   sudo systemctl enable starmap-backend
   ```

---

## 第四步：配置 Nginx 统一网关

最后一步，我们用 Nginx 把前端页面和后端接口“缝合”在一起，对外只暴露 80 或 443（HTTPS）端口。

编辑 Nginx 配置文件（如 `/etc/nginx/sites-available/starmap`）：

```nginx
server {
    listen 80;
    server_name yourdomain.com; # 替换为你的真实域名或服务器 IP

    # 1. 拦截前端静态文件请求
    location / {
        root /var/www/starmap/frontend/dist;  # 指向前端打包后的 dist 目录
        index index.html;
        try_files $uri $uri/ /index.html;     # 解决 React Router 单页应用刷新 404 问题
    }

    # 2. 拦截所有以 /api 开头的请求，代理转发给 Python 后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000;     # 转发到 FastAPI 的运行端口
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

保存后，验证配置并重启 Nginx：
```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 💡 扩展：其他极客部署方案 (选读)

如果您觉得传统的服务器配置过程较为繁琐，现代前后端分离项目还常常采用以下两种方案：

### 1. Docker 容器化部署（强烈推荐）
编写两个 `Dockerfile`（分别针对前端和后端）以及一个 `docker-compose.yml` 文件。将代码拉取到服务器后，只需执行一行命令 `docker-compose up -d`，所有的环境、依赖和网络反向代理将自动配置完成。这极大提升了项目的可移植性。

### 2. Serverless / PaaS 懒人部署
- **前端托管**：将 GitHub 仓库直接授权给 **Vercel** 或 **Netlify**，每次推送代码后，平台会自动执行打包并分发到全球 CDN，提供极其快速的访问体验。
- **后端托管**：将 FastAPI 服务部署到 **Render**、**Railway** 或 **Zeabur** 上。
*这种方案的优点在于完全不需要购买和维护 Linux 虚拟机实例。*
